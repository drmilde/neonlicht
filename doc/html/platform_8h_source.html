67 // Some operating systems, such as Solaris
68 // (see http://developers.sun.com/solaris/articles/stdio_256.html),
69 // have an implementation of <stdio.h> in which fopen() fails if
70 // the file descriptor used is > 255. This suggests that config4Cpp
71 // should use open() instead of fopen(). However, fopen() is part of
72 // the standard C library and is guaranteed to be available everywhere,
73 // but open() is UNIX-specific and may not be available on some
74 // (non-UNIX) systems. The workaround is to define a class that provides
75 // an abstraction layer around either fopen() or open(). The
76 // functionality of this class is driven by the needs of the Lex class,
77 // which needs only to open a file for reading and then read it one
78 // byte at a time.
79 //----------------------------------------------------------------------
